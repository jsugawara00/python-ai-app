## 1. アプリ概要

Google Gemini API を利用した **8 種類の AI ライティングツール**を提供する Streamlit シングルページアプリケーション。ブラウザから直接利用でき、生成結果は PDF・Word・Excel 形式でダウンロード可能。

### 技術スタック

| 項目 | 内容 |
|------|------|
| フレームワーク | Streamlit >= 1.28.0 |
| AI モデル | Google Gemini 3.5 Flash（`gemini-3.5-flash`） |
| PDF 生成 | reportlab >= 4.0.0（CIDフォント HeiseiKakuGo-W5） |
| Word 生成 | python-docx >= 1.0.0 |
| Excel 生成 | openpyxl >= 3.1.0 |
| 言語 | Python 3.x |
| 構成 | シングルファイル（`app.py`） |

### セットアップ

```bash
# 1. 依存パッケージのインストール
pip install -r requirements.txt

# 2. 環境変数の設定
# .env.example をコピーして .env を作成し GEMINI_API_KEY を設定する

# 3. 起動
streamlit run app.py
# → ブラウザで http://localhost:8501 が自動起動。ファイル保存でホットリロード。
```

### API キー入力方法

- **方法 A**：アプリ起動後、サイドバーの「Gemini API Key」欄に直接入力
- **方法 B**：`.env` ファイルに `GEMINI_API_KEY=your_key` を記載して起動

## 2. アーキテクチャ

### 2-1. ルーティング（TOOLS 辞書）

`TOOLS` 辞書がサイドバーナビゲーションとページ関数を一元管理する。

```python
TOOLS = {
    "📊 営業報告書作成": ("sales_report", page_sales_report),
    "📝 ブログ記事執筆": ("blog", page_blog),
    "📧 メール返信作成": ("email", page_email),
    ...
}
```

サイドバーの `st.radio` で選択されたラベルをもとに `page_func()` を呼び出す。  
**新しいツールを追加する場合は `page_xxx()` を実装して `TOOLS` に登録するだけ。**

### 2-2. AI 呼び出し（`generate` 関数）

`generate(prompt, temperature, system_instruction)` が唯一の AI 呼び出し口。

| パラメータ | 内容 |
|-----------|------|
| モデル | `gemini-3.5-flash`（固定） |
| temperature | デフォルト `0.7` / 精度重視ツールは `0.3` または `0.2` |
| エラー処理 | 例外をキャッチして日本語メッセージを返す（呼び出し元の処理不要） |

### 2-3. 初期化フロー

`configure_gemini()` が `main()` 先頭で一度だけ呼ばれ、APIキーの有無を検証する。  
戻り値 `is_configured`（bool）をサイドバーのステータス表示とページ上部の警告に使用。  
APIキー未設定でも画面は表示されるが、生成ボタン押下時にエラーメッセージを返す。

### 2-4. スタイリング

- カスタム CSS は `main()` 内の `st.markdown(..., unsafe_allow_html=True)` に集約
- サイドバー幅は `min-width:250px / max-width:300px` で固定
- サイドバーのタイトルヘッダーはカスタム HTML で実装（`st.title()` 未使用）

## 4. 営業報告書作成ツール 詳細仕様

### 4-1. 入力項目

**表題**（フォーム外・リアルタイムプレビュー）

- 種類：ラジオボタン（営業 / 出張 / 訪問 / その他）
- その他選択時：テキスト入力で任意ラベルを指定
- `「〇〇報告書」` の形式で表題をリアルタイムプレビュー

**基本情報**（フォーム内・2 列レイアウト）

| 左列 | 右列 |
|------|------|
| 🏢 訪問先 | 👥 自社担当者 |
| 📅 訪問日時 | ✏️ 報告者 |
| 👤 先方対応者 | 📆 報告日 |

**本文セクション**（フォーム内・テキストエリア）

1. 訪問目的
2. 訪問の実施結果
3. 当社の課題
4. 今後の展望

### 4-2. AI 生成仕様

- `temperature = 0.3`（精度重視）
- `system_instruction` でビジネス文書専門家として動作を指定
- 未入力項目は文脈から合理的に補完
- 出力は `【セクション名】` 形式でパース後、入力値でフォールバック補完

### 4-3. ダウンロード形式

**PDF（`sales_report.pdf`）**

- A4 / 余白 20mm / フォント HeiseiKakuGo-W5
- 右上に報告日・報告者（右寄せ、列幅 60mm）
- タイトル中央寄せ → 区切り線（ネイビー 2pt）
- 基本情報：2 行 × 4 列テーブル（ラベル幅 65% / 値幅 135%）
- セルはみ出し対策：`wordWrap="CJK"` + `splitLongWords=True`
- セクションヘッダー：色分け背景（紺 / 青 / 赤 / 緑）

**Excel（`sales_report.xlsx`）**

- 列幅：A=14 / B=30 / C=14 / D=30
- 右上 2 行（C:D 結合）に報告日・報告者
- タイトル行：A:D 結合、高さ 38pt、中央寄せ
- 基本情報：2 行（訪問先/訪問日時、先方対応者/自社担当者）
- セクションヘッダー + 折り返し対応コンテンツ行

**Word（`sales_report.docx`）**

- A4 / 余白：上下 2cm / 左 2.5cm / 右 2.5cm
- 右上に報告日・報告者（右寄せ）
- タイトル中央・下罫線（XML `pBdr`）
- 基本情報：リスト形式（`訪問先：xxx`、`訪問日時：xxx` …）
- セクション見出し（heading level=2）＋本文

### 4-4. フォーム実装の注意事項

`st.form()` + `st.form_submit_button()` を使用している。

> **理由：** `st.button()` はクリック時にスクリプト全体を再実行するため、テキスト入力の値がリセットされ基本情報がブランクになる問題があった。`st.form()` はフォーム送信時に全ウィジェット値をアトミックに session_state へコミットしてから再実行するため、フォームの戻り値を直接使用することで確実に値を取得できる。

## 5. 汎用ダウンロードヘルパー

### `_to_pdf_bytes(text, image_bytes, image_position)`

マークダウン形式テキストを A4 PDF に変換。`# / ## / ###` 見出しに対応。  
`image_bytes` が指定された場合、`image_position`（文章の先頭 / 最初の見出しの後 / 文章の末尾）に画像を挿入。

### `_to_docx_bytes(text, image_bytes, image_position)`

マークダウン形式テキストを Word ファイルに変換。見出しレベルと画像挿入位置に対応。

## 6. 新ツールの追加手順

```python
# 1. page_xxx() 関数を実装
def page_xxx():
    st.title("🎯 ツール名")
    # ... 入力UI、AI呼び出し、結果表示 ...

# 2. TOOLS 辞書に追加
TOOLS = {
    ...
    "🎯 ツール名": ("key", page_xxx),
}
# → それだけでサイドバーに表示される
```

### 設計ルール

- 各 `page_*` 関数は独立（関数間で状態を共有しない）
- プロンプトは各 `page_*` 関数内にハードコード
- `st.session_state` はホットリロードでリセットされる点に注意
- 精度重視のツール（校正・要約・翻訳・報告書）は `temperature=0.3` 以下を使用
- AI 呼び出しは `generate()` のみ使用し、直接 `genai` を呼ばない
